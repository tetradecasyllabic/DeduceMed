Core vs. Optional Probability

How it works: Diseases now split symptoms into Core (Expected) and Optional (Has a Chance). If you check an optional symptom, the probability goes up! If you do *not* check it, no penalty occurs. However, checking unrelated symptoms (like "Skin Rash" for Strep Throat) triggers the **Noise Penalty** and drops its probability.

Separate Files Architecture Blueprint

Ready to deploy your 80+ disease sandbox to GitHub Pages? Here is how to segment this workspace into a standard web structure.

index.html

1. Document Markup

Create `index.html`. Strip the `styles` and `script` elements, link them inside the `` tag, and paste the responsive body frame. 

<link rel="stylesheet" href="styles.css">
<script src="app.js" defer></script>
styles.css

2. Styling Sheet

Create `styles.css`. Paste the customized scrollbars, animations, responsive slider styling, and card transitions from our stylesheet block.

app.js

3. Clinical Application Logic

Create `app.js`. Paste the disease dictionary, category lists, search logic, dynamic slider hooks, and the penalty loop algorithm.

How to Host This on GitHub Pages

Upload all three files to your GitHub repository. Go to your repo Settings > Pages, select the branch as main, save, and your application will instantly go live globally.
